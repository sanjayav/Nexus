# Internationalisation (i18n)

Nexus uses [react-i18next](https://react.i18next.com/) for localisation. The
infrastructure is wired up; full string coverage is rolled out incrementally.

## Architecture

- `src/i18n/index.ts` — boots i18next with browser language detection,
  registers all locales, persists user choice to `localStorage` under
  `aeiforo_lng`.
- `src/i18n/<lang>.json` — flat-key resource bundle per language.
- `src/i18n/th.notes.json` — translator notes for terms left in English.
- `src/main.tsx` — imports `./i18n` for side effects before `<App />` renders.
- Language switcher — globe-icon dropdown in the topbar
  (`src/design-system/components/TopBar.tsx`, `LanguageSwitcher`).

## Source of truth

**English (`en.json`) is canonical.** Every key MUST exist in `en.json`. Other
locales are best-effort and fall back to English when a key is missing.

## Adding a new locale

1. Pick a [BCP 47 / ISO 639-1 code](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes)
   (e.g. `de`, `fr`, `ja`).
2. Create `src/i18n/<code>.json` mirroring every key in `en.json`.
3. Register it in `src/i18n/index.ts`:

   ```ts
   import de from './de.json'
   // ...
   resources: {
     en: { translation: en },
     th: { translation: th },
     de: { translation: de },
   },
   supportedLngs: ['en', 'th', 'de'],
   ```

4. Add the locale to the `LOCALES` array in `TopBar.tsx`:

   ```ts
   const LOCALES = [
     { code: 'en', label: 'English' },
     { code: 'th', label: 'ไทย' },
     { code: 'de', label: 'Deutsch' },
   ]
   ```

5. If any terms are left untranslated (jargon, acronyms), document them in
   `src/i18n/<code>.notes.json` so a native reviewer can finalise.

## Usage in components

```tsx
import { useTranslation } from 'react-i18next'

function Login() {
  const { t } = useTranslation()
  return <button>{t('login.signIn')}</button>
}
```

Interpolation:

```tsx
t('myTasks.dueThisWeek', { count: 5 }) // → "5 due this week"
```

## Locale review checklist

When adding/updating strings:

- [ ] Add the key + English value to `en.json` first.
- [ ] Add a translated value (or English fallback + TODO note) in each other
      locale file.
- [ ] Avoid baking grammar/word-order into JS string concatenation — use
      interpolation placeholders so translators can reorder.
- [ ] Numbers/dates with locale formatting: use `Intl.NumberFormat` /
      `Intl.DateTimeFormat` with `i18n.language`.
